# インシデント対応ガイド（Incident Response Guide）
**プロジェクト**: Appsuite専用運用管理システム  
**作成日**: 2026-02-25  

## 1. インシデント重大度定義
| 重大度 | 定義 | 例 | 対応時間 |
|-------|------|-----|---------|
| P1（緊急） | サービス全停止 | サーバーダウン、全ユーザーアクセス不可 | 1時間以内 |
| P2（高） | 主要機能停止 | 認証不可、データ保存不可 | 4時間以内 |
| P3（中） | 一部機能障害 | 特定ページ表示不可 | 翌営業日 |
| P4（低） | 軽微な問題 | 表示崩れ、誤字 | 1週間以内 |

## 2. エスカレーションフロー
```
発生
  ↓
運用担当者（初動対応・30分）
  ↓ P1/P2の場合
システム管理者（原因調査・1時間）
  ↓ 解決困難な場合
開発チーム（緊急修正対応）
  ↓
マネージャー（ステークホルダー報告）
```

## 3. よくあるインシデントと対処法

### ケース1: ログイン画面が表示されない
```
確認項目:
  1. サーバーの稼働確認: systemctl status nginx
  2. ファイルの存在確認: ls /var/www/appsuite/index.html
  3. Nginxエラーログ確認: tail -n 100 /var/log/nginx/error.log
対処:
  - Nginx再起動: systemctl restart nginx
  - ファイル権限確認: chmod -R 755 /var/www/appsuite
```

### ケース2: DeskNet's Neo APIとの接続エラー
```
確認項目:
  1. ブラウザコンソールのエラーメッセージ確認
  2. DeskNet's Neoサービス稼働確認
  3. APIエンドポイントURL・APIキーの確認
対処:
  - 設定画面でAPIエンドポイントを再設定
  - DeskNet's Neo管理者に連絡
```

### ケース3: データが消えた（localStorage/IndexedDB）
```
確認項目:
  1. ブラウザのキャッシュクリア有無確認
  2. 別ブラウザ/PCからアクセス確認
  3. バックアップデータ確認
対処:
  - バックアップから復元
  - 自動バックアップ設定の確認・修正
  - ユーザーへのデータ入力依頼
```
