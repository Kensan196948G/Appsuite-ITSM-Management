## 1. プロジェクト概要

### 1.1 目的

AppSuite ITSM管理システムは、DeskNet's Neo AppSuiteで作成された業務アプリケーションを一元管理し、ITサービス管理（ITSM）のベストプラクティスに基づいた運用を実現することを目的とします。

### 1.2 スコープ

**対象範囲**:
- ユーザー管理（アカウント・権限）
- アプリケーション管理（稼働状況・カテゴリ）
- インシデント管理（障害対応・追跡）
- 変更管理（承認フロー・スケジュール）
- 監査ログ（操作履歴・エクスポート）
- システム設定（API・セキュリティ・通知）

**対象外**:
- DeskNet's Neo本体の管理機能
- メールサーバー・通知基盤の構築
- 外部認証システム（SAML/OAuth）の実装

### 1.3 KGI/KPI

| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| インシデント対応時間 | 平均2時間以内 | ステータス変更時刻の差分 |
| 変更要求承認率 | 80%以上 | 承認済/申請数 |
| システム可用性 | 99.5%以上 | 稼働時間/計画時間 |
| ユーザー満足度 | 4.0/5.0以上 | 定期アンケート |

## 4. 非機能要件

### 4.1 性能要件

| 項目 | 要件 |
|------|------|
| 初期表示 | 3秒以内 |
| 画面遷移 | 1秒以内 |
| 検索応答 | 500ms以内 |

### 4.2 セキュリティ要件

| 項目 | 対策 |
|------|------|
| XSS | 入力エスケープ（escapeHtml） |
| 認証 | SHA-256パスワードハッシュ |
| セッション | 30分タイムアウト |
| ブルートフォース | 5回失敗で15分ロック |

### 4.3 可用性要件

| 項目 | 要件 |
|------|------|
| 稼働時間 | 99.5%（月間3.6時間以内の停止） |
| バックアップ | 日次（localStorage export） |
| リカバリ | 1時間以内 |

## 5. 制約事項

1. **ブラウザ依存**: localStorage使用のため、ブラウザ・デバイス間でのデータ共有不可
2. **容量制限**: localStorage上限は5-10MB
3. **オフライン動作**: 外部APIへのアクセスは不可
4. **マルチユーザー**: 同時編集時の競合検知機能なし
