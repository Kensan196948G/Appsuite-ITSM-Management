| 原則 | 説明 |
|------|------|
| **シンプル** | 情報密度を適切に、視覚的ノイズを最小化 |

| **効率性** | 少ないクリックで目的達成 |
| **フィードバック** | 操作結果を明確に表示 |

## 5. レスポンシブ対応

### 5.1 ブレークポイント

| ブレークポイント | 画面サイズ | レイアウト変更 |
|----------------|-----------|---------------|
| **Desktop** | 1024px以上 | 通常レイアウト |
| **Tablet** | 768px-1023px | ナビゲーション折りたたみ |
| **Mobile** | 768px未満 | サポート外（将来対応） |

### 5.2 Tablet対応

```css
@media (max-width: 1023px) {
  /* ナビゲーションをハンバーガーメニュー化 */
  .sidebar {
    position: fixed;
    left: -220px;
    transition: left 0.3s;
  }

  .sidebar.active {
    left: 0;
  }

  /* コンテンツエリアを全幅に */
  .main-content {
    margin-left: 0;
  }
}
```
