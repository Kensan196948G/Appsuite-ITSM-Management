# テスト計画書（Test Plan）
**プロジェクト**: Appsuite専用運用管理システム  
**作成日**: 2026-02-25  

## 1. テスト方針
本システムはフロントエンドのみのSPAであるため、以下のテスト戦略を採用する。

| テストレベル | ツール | 対象 |
|-----------|-------|------|
| 単体テスト | Vitest / Jest | ユーティリティ関数、API処理 |
| 統合テスト | Playwright | コンポーネント間連携 |
| E2Eテスト | Playwright | 主要業務フロー |
| 手動テスト | テストケース表 | UAT（ユーザー受入テスト） |
| セキュリティテスト | OWASP ZAP | 脆弱性スキャン |

## 2. テスト対象機能
| 機能 | 単体 | 統合 | E2E | UAT |
|------|------|------|-----|-----|
| 認証・ログイン | ✅ | ✅ | ✅ | ✅ |
| ダッシュボード | - | ✅ | ✅ | ✅ |
| ユーザー管理 | ✅ | ✅ | ✅ | ✅ |
| アプリ管理 | ✅ | ✅ | ✅ | ✅ |
| インシデント管理 | ✅ | ✅ | ✅ | ✅ |
| 変更管理 | ✅ | ✅ | ✅ | ✅ |
| 監査ログ | ✅ | ✅ | ✅ | ✅ |
| システム設定 | ✅ | ✅ | ✅ | ✅ |

## 3. テスト環境
| 環境 | URL | 用途 |
|------|-----|------|
| 開発環境 | http://localhost:3000 | 開発者テスト |
| ステージング | https://staging.example.com | 統合・UAT |
| 本番 | https://appsuite.example.com | 本番確認 |

## 4. テスト完了基準
- 単体テストカバレッジ: 80%以上
- E2Eテスト: 全主要フロー合格
- セキュリティテスト: 高・中リスク脆弱性ゼロ
- パフォーマンス: LCP 2.5秒以内
- アクセシビリティ: Lighthouse 90点以上

## 5. 不具合管理
| 重大度 | 対応期限 | 基準 |
|--------|---------|------|
| Critical | 24時間以内 | システム停止・データ損失 |
| High | 3営業日以内 | 主要機能の動作不全 |
| Medium | 1週間以内 | 機能の一部不具合 |
| Low | 次リリース | 軽微な表示崩れ等 |
