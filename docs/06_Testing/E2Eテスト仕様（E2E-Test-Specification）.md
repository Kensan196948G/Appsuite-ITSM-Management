# E2Eテスト仕様（E2E Test Specification）
**プロジェクト**: Appsuite専用運用管理システム  
**使用ツール**: Playwright  
**作成日**: 2026-02-25  

## 1. テストシナリオ一覧

### TC-001: ログイン・ログアウト
```
前提: ユーザーアカウントが存在する
手順:
  1. ログインページにアクセス
  2. ユーザーID・パスワードを入力
  3. ログインボタンをクリック
  4. ダッシュボードが表示されることを確認
  5. ヘッダーのユーザーアイコンをクリック
  6. ログアウトを選択
  7. ログインページに遷移することを確認
期待結果: ログイン成功→ダッシュボード表示、ログアウト後→ログインページ遷移
```

### TC-010: ユーザー作成フロー
```
前提: 管理者でログイン済み
手順:
  1. ユーザー管理ページに遷移
  2. 「新規ユーザー追加」ボタンをクリック
  3. 氏名・メール・権限を入力・選択
  4. 「保存」ボタンをクリック
  5. 一覧に新ユーザーが表示されることを確認
  6. 監査ログに記録されることを確認
期待結果: ユーザー作成成功、監査ログに CREATE USER が記録
```

### TC-030: インシデント登録・解決フロー
```
前提: 一般ユーザーでログイン済み
手順:
  1. インシデント管理ページに遷移
  2. 「新規インシデント」ボタンをクリック
  3. タイトル・説明・優先度を入力
  4. 「登録」ボタンをクリック
  5. 一覧に表示されることを確認
  6. インシデントを開き「対応中」に変更
  7. 解決後「解決済み」に変更
  8. 「クローズ」に変更
期待結果: ステータス遷移が正常に行われる
```

## 2. テスト実行コマンド
```bash
# 全テスト実行
npx playwright test

# 特定テストのみ実行
npx playwright test tests/auth.spec.js

# UIモードで実行（デバッグ用）
npx playwright test --ui

# レポート生成
npx playwright show-report
```
