## 1. デザイン原則

### 1.1 5つのコア原則

| 原則 | 説明 |
|------|------|
| **シンプリシティ** | 必要な情報・機能のみを表示し、認知負荷を最小化する |
| **一貫性** | 同じ操作・情報には常に同じUI・言葉を使う |
| **フィードバック** | すべての操作に対して即座に視覚的応答を返す |
| **柔軟性** | 初心者からエキスパートまで効率よく使えるUI |
| **寛容性** | 間違いを防止し、間違えても回復しやすい設計 |

## 6. コンポーネントUX仕様

### 6.1 ボタン使用ガイドライン

| シチュエーション | ボタン種別 |
|--------------|---------|
| 最も重要なアクション（1画面に1つ） | Primary |
| 2番目に重要なアクション | Secondary |
| 破壊的操作（削除等） | Danger |
| キャンセル・戻る | Ghost / Secondary |
| テキストリンク的操作 | Link |

**ボタン配置規則:**
- 確認ダイアログ: 左に「キャンセル」、右に「実行（Danger/Primary）」
- フォーム: 右寄せ、左から「キャンセル」「保存」

### 6.2 フォームUXガイドライン

| 原則 | 実装 |
|------|------|
| 入力中のバリデーション | blur（フォーカス離脱）時に実行、onChangeは避ける |
| エラー表示位置 | 対象フィールドの直下 |
| 必須フィールド | アスタリスク（*）でマーク、凡例を上部に表示 |
| 自動保存 | 長いフォームは30秒ごとにドラフト保存 |
| タブオーダー | 視覚的レイアウト順と一致させる |

### 6.3 データテーブルUXガイドライン

| 項目 | 仕様 |
|------|-----|
| ゼブラストライプ | 50行以上の場合に有効化 |
| ホバーハイライト | 常時有効 |
| 行クリック | 詳細ページへ遷移（クリッカブルな視覚的フィードバック） |
| ソートアイコン | ソート中のカラムには矢印アイコンを表示 |
| 選択状態 | チェックボックスと行全体のハイライト |

## 7. インタラクション設計

### 7.1 アニメーション

| 種別 | 時間 | イージング | 用途 |
|------|------|----------|------|
| マイクロインタラクション | 100-200ms | ease-out | ボタンクリック、ホバー |
| UI要素の表示/非表示 | 200-300ms | ease-in-out | ドロップダウン、トースト |
| ページ遷移 | 200ms | ease | ルート変更 |
| モーダル | 250ms | ease-out | モーダル開閉 |
| ローディング | 無限 | linear | スピナー回転 |

**アニメーション方針:**
- `prefers-reduced-motion` メディアクエリに対応（アクセシビリティ）
- 装飾的なアニメーションはこの設定が有効な場合は無効化

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 7.2 ローディングパターン

| データ量 | ローディング表示 |
|---------|--------------|
| 初回ページロード | スケルトンスクリーン |
| テーブルデータ取得 | スケルトンスクリーン（行数維持） |
| ボタン操作後 | ボタン内スピナー + 無効化 |
| 全画面ブロッキング | オーバーレイスピナー（最終手段） |
| バックグラウンド更新 | インジケーターなし（UX優先） |

### 7.3 空状態・エラー状態

## 8. ライティングガイドライン

### 8.1 UI テキストの原則

| 原則 | 悪い例 | 良い例 |
|------|-------|-------|
| 能動態を使う | 「データは削除されました」 | 「データを削除しました」 |
| 簡潔にする | 「次のステップを続行するためにここをクリックしてください」 | 「次へ」 |
| ポジティブに表現する | 「このフォームを送信しないでください」 | 「保存前に確認してください」 |
| ユーザー目線で書く | 「エラー 422：バリデーション失敗」 | 「入力内容に誤りがあります」 |

### 8.2 ボタンラベル規則

| アクション種別 | ラベル例 |
|------------|---------|
| 作成系 | 「作成」「追加」「登録」「新規作成」 |
| 保存系 | 「保存」「更新」「変更を保存」 |
| 削除系 | 「削除」「削除する」（「はい」は避ける） |
| 閉じる系 | 「キャンセル」「閉じる」「戻る」 |
| 確認系 | 「確認」「OK」（破壊的操作には使わない） |
