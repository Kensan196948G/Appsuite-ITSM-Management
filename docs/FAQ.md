## 1. ログイン・認証

### Q1-1. 初回ログインのパスワードがわかりません

**A**: デフォルトの管理者アカウントは以下です:
- ユーザー名: `admin`
- パスワード: 初回セットアップ時に設定したもの

一般ユーザーの初回パスワードは、システム管理者から個別に通知されます。

### Q1-2. パスワードを忘れました

**A**: システム管理者に連絡してパスワードリセットを依頼してください。
セキュリティのため、本人確認が必要です。

**連絡先**:
- メール: admin@example.com
- 電話: 内線xxxx

### Q1-3. 「セッションがタイムアウトしました」と表示される

**A**: 30分間操作がない場合、セキュリティのため自動的にログアウトされます。
再度ログインして利用を再開してください。

セッションタイムアウト時間は、システム設定で変更可能です（管理者のみ）。

### Q1-4. ログインボタンを押してもログインできない

**A**: 以下を確認してください:

1. **ユーザー名・パスワードが正しいか**
   - 大文字・小文字を区別します
   - スペースが含まれていないか確認

2. **アカウントが有効か**
   - 管理者にアカウントステータスを確認してもらう

3. **ブラウザのJavaScriptが有効か**
   - ブラウザ設定でJavaScriptを有効化

4. **ネットワーク接続**
   - インターネット接続を確認

### Q1-5. HTTPSで「安全ではありません」と表示される

**A**: SSL証明書に問題がある可能性があります。
システム管理者に連絡してください。

**一時的な対処**（開発環境のみ）:
- 証明書エラーを承認して続行
- HTTP（非暗号化）でアクセス: `http://localhost:8888/`

⚠️ **注意**: 本番環境では必ずHTTPSを使用してください。

## 2. ユーザー管理

### Q2-1. 新しいユーザーを追加したいのですが、どうすればよいですか？

**A**: 管理者権限が必要です。以下の手順で追加できます:

1. サイドメニュー→「ユーザー管理」
2. 「新規ユーザー」ボタンをクリック
3. 必須項目を入力:
   - ユーザー名
   - メールアドレス
   - 部署
   - 権限（管理者/ユーザー）
4. 「保存」をクリック

一般ユーザーはユーザー追加ができません。管理者に依頼してください。

### Q2-2. ユーザーの権限を変更したい

**A**: 管理者のみが変更可能です。

1. ユーザー管理画面で対象ユーザーの「✏️」ボタンをクリック
2. 権限を「管理者」または「ユーザー」に変更
3. 「保存」をクリック

⚠️ **重要**: 権限変更は上長の承認を得てから実施してください。

### Q2-3. 退職者のアカウントはどうすればよいですか？

**A**: 以下の2つの方法があります:

**方法1: ステータスを無効に変更（推奨）**
1. ユーザーを編集
2. ステータスを「無効」に変更
3. 保存

この方法では、過去の操作履歴が保持されます。

**方法2: 削除**
1. ユーザーの「🗑️」ボタンをクリック
2. 確認ダイアログで「OK」

⚠️ **注意**: 削除すると復元できません。通常は無効化を推奨します。

### Q2-4. 自分の情報を変更したい

**A**: 一般ユーザーは自分のパスワードのみ変更可能です。
その他の情報（メールアドレス、部署等）は管理者に依頼してください。

**パスワード変更手順**:
1. ヘッダーの自分のユーザー名をクリック
2. 「パスワード変更」を選択
3. 現在のパスワードと新しいパスワードを入力
4. 「変更」をクリック

## 3. インシデント管理

### Q3-1. インシデントと変更要求の違いは何ですか？

**A**:

| 項目 | インシデント | 変更要求 |
|------|-------------|---------|
| **定義** | 既に発生している問題・障害 | 新機能追加・改善の申請 |
| **目的** | 迅速な問題解決 | 計画的な改善 |
| **例** | ログインできない、エラーが出る | 新機能を追加したい、UIを改善したい |
| **対応** | 即座に対応 | 承認後に計画的に対応 |

### Q3-2. インシデントの優先度はどう決めればよいですか？

**A**: 以下の基準で判断してください:

| 優先度 | 判断基準 | 対応目標 |
|--------|---------|---------|
| **高** | ・業務が完全に停止している<br>・多数のユーザーに影響<br>・データ損失のリスク | 4時間以内 |
| **中** | ・業務に支障があるが継続可能<br>・一部のユーザーに影響<br>・回避策がある | 24時間以内 |
| **低** | ・軽微な問題<br>・影響が限定的<br>・改善要望レベル | 72時間以内 |

**迷ったら**: 高めに設定し、担当者が調整します。

### Q3-3. インシデントを登録したのに対応されません

**A**: 以下を確認してください:

1. **担当者がアサインされているか**
   - 未アサインの場合、管理者に連絡

2. **優先度が適切か**
   - 緊急の場合、優先度を「高」に設定

3. **ステータスが「オープン」か**
   - 誤って「クローズ」していないか確認

4. **連絡先が正しいか**
   - 報告者の連絡先が正しく記載されているか

**緊急の場合**: 管理者に直接連絡してください。

### Q3-4. インシデントを間違って登録しました

**A**: 以下の2つの方法があります:

**方法1: 編集して修正**
1. インシデントの「✏️」ボタンをクリック
2. 内容を修正
3. 「保存」をクリック

**方法2: 削除**
1. インシデントの「🗑️」ボタンをクリック
2. 確認ダイアログで「OK」

⚠️ **注意**: 削除は取り消せません。通常は編集を推奨します。

### Q3-5. インシデントがクローズされたのに問題が解決していません

**A**: 以下の手順で再オープンしてください:

1. クローズされたインシデントを開く
2. 「✏️」ボタンで編集
3. ステータスを「対応中」に戻す
4. コメント欄に「問題が解決していない」旨を記載
5. 「保存」をクリック

または、新しいインシデントとして再登録することもできます。

## 4. 変更管理

### Q4-1. 変更要求を作成しましたが、承認されません

**A**: 承認には時間がかかることがあります。以下を確認してください:

1. **ステータスが「承認待ち」か**
   - 「下書き」のままだと承認されません
   - 「承認待ち」に変更してください

2. **変更内容が明確か**
   - 「なぜ必要か」「どんな効果があるか」を明記
   - 不明瞭な場合、却下または差し戻しされます

3. **承認者に通知されているか**
   - 管理者に確認してください

**通常の承認期間**: 2-5営業日

### Q4-2. 緊急の変更はどうすればよいですか？

**A**: 以下の手順で緊急変更を申請してください:

1. 変更要求を作成
2. タイトルに「【緊急】」を付ける
3. 説明欄に緊急性の理由を明記
4. 管理者に直接連絡（メール・電話）

緊急変更は、通常よりも早く承認されます（数時間〜1営業日）。

### Q4-3. 変更要求が却下されました

**A**: 却下理由を確認し、以下の対応を検討してください:

1. **却下理由を確認**
   - コメント欄に却下理由が記載されています

2. **再申請**
   - 理由を踏まえて内容を修正
   - 新しい変更要求として再申請

3. **相談**
   - 理由が不明な場合、管理者に相談

**よくある却下理由**:
- 費用対効果が低い
- 技術的に実現困難
- 他の変更と競合
- 説明が不十分

## 5. システム設定

### Q5-1. API接続が「未接続」のままです

**A**: 管理者のみが設定可能です。以下の手順で設定してください:

1. システム設定→API接続タブ
2. DeskNet's Neo API情報を入力:
   - API URL: `https://your-domain.desknets.com/cgi-bin/dneo/zap.cgi`
   - APIキー: DeskNet's Neo管理画面から取得
   - 認証方式: Bearer Token（推奨）
3. 「接続テスト」をクリック
4. 「保存」をクリック

**接続テスト失敗時**:
- URL、APIキー、認証方式を確認
- DeskNet's Neoのサーバーが稼働しているか確認
- ネットワーク接続を確認

### Q5-2. パスワードポリシーを変更したい

**A**: 管理者のみが変更可能です。

1. システム設定→セキュリティタブ
2. パスワードポリシーの項目を変更:
   - 最小文字数（6-32文字）
   - 大文字必須（有効/無効）
   - 数字必須（有効/無効）
   - 特殊文字必須（有効/無効）
3. 「保存」をクリック

⚠️ **注意**: セキュリティのため、以下を推奨します:
- 最小8文字以上
- 大文字・数字必須

### Q5-3. セッションタイムアウト時間を変更したい

**A**: 管理者のみが変更可能です。

1. システム設定→セキュリティタブ
2. セッションタイムアウト時間を変更（分単位）
3. 「保存」をクリック

**推奨値**: 30分（デフォルト）

## 6. トラブルシューティング

### Q6-1. 画面が表示されない・真っ白になる

**A**: 以下を順番に試してください:

1. **ブラウザのキャッシュをクリア**
   - Chrome: Ctrl + Shift + Delete
   - 「キャッシュされた画像とファイル」を選択して削除

2. **強制再読み込み**
   - Ctrl + Shift + R（Windows）
   - Cmd + Shift + R（Mac）

3. **JavaScriptが有効か確認**
   - ブラウザ設定でJavaScriptを有効化

4. **別のブラウザで試す**
   - Chrome、Firefox、Edge、Safariで確認

5. **システム管理者に連絡**

### Q6-2. データが保存されない

**A**: 以下を確認してください:

1. **localStorage容量**
   - ブラウザ開発者ツール（F12）→Application→Local Storage
   - 5MB以上使用している場合、古いデータを削除

2. **シークレットモード・プライベートブラウズ**
   - 通常モードで使用してください

3. **ブラウザのストレージ設定**
   - Cookie・ストレージが許可されているか確認

4. **入力エラー**
   - 必須項目が全て入力されているか確認
   - エラーメッセージがないか確認

### Q6-3. 「403 Forbidden」エラーが表示される

**A**: 権限がない操作を実行しようとしています。

**原因**:
- 一般ユーザーが管理者機能にアクセスしようとした
- アカウントが無効になっている

**対処**:
1. 必要な権限を管理者に確認
2. アカウントステータスを確認してもらう

### Q6-4. 画面が遅い・重い

**A**: 以下を試してください:

1. **ブラウザキャッシュをクリア**
   - Ctrl + Shift + Delete

2. **データ量を削減**
   - 古いログを削除・エクスポート
   - システム設定→監査ログ→古いログを削除

3. **ブラウザ拡張機能を無効化**
   - 広告ブロッカー等を一時的に無効化

4. **ネットワーク速度を確認**
   - 速度測定ツールで確認

5. **別のブラウザで試す**

### Q6-5. エクスポートしたCSVファイルが文字化けする

**A**: Excelで開く場合、文字コードの問題が発生することがあります。

**対処法1: Excelインポート機能を使用**
1. Excelを開く
2. データ→テキストまたはCSVから
3. ファイルを選択
4. 文字コードを「UTF-8」に設定
5. インポート

**対処法2: メモ帳で確認**
- メモ帳（UTF-8対応エディタ）で開いて内容を確認

## 7. データ管理・バックアップ

### Q7-1. データはどこに保存されていますか？

**A**: ブラウザのlocalStorage（ブラウザ内蔵ストレージ）に保存されています。

**特徴**:
- サーバーには送信されない
- ブラウザごとに独立
- 容量制限: 約5-10MB

**注意点**:
- ブラウザのデータをクリアすると消えます
- 定期的にバックアップを取ってください

### Q7-2. バックアップはどのくらいの頻度で取るべきですか？

**A**: 以下の頻度を推奨します:

| バックアップ種別 | 頻度 | タイミング |
|----------------|------|-----------|
| **日次** | 毎日 | 業務終了時 |
| **週次** | 週1回 | 金曜日 |
| **月次** | 月1回 | 月末 |
| **重要変更前** | 都度 | 大きな変更を加える前 |

**自動バックアップ**: 管理者が設定可能（運用マニュアル参照）

### Q7-3. バックアップファイルはどこに保存すればよいですか？

**A**: 以下のいずれかに保存してください:

1. **会社の共有ドライブ**（推奨）
   - アクセス権限を適切に設定

2. **個人のローカルフォルダ**
   - 定期的に外部ストレージにコピー

3. **クラウドストレージ**
   - Google Drive、OneDrive等
   - セキュリティポリシーを確認

**命名規則**:
```
appsuite_backup_YYYYMMDD_HHMMSS.json
例: appsuite_backup_20260204_143025.json
```

### Q7-4. 誤ってデータを削除してしまいました

**A**: バックアップから復元できます。

**手順**:
1. システム設定→バックアップタブ
2. 「バックアップファイル」で最新のバックアップを選択
3. 「リストア実行」をクリック
4. 確認ダイアログで「OK」
5. ページが自動リロード

⚠️ **注意**: リストアは現在のデータを全て上書きします。

**バックアップがない場合**: 復元できません。定期的にバックアップを取ってください。

### Q7-5. 他の人とデータを共有したい

**A**: 現在のバージョンでは、ブラウザごとに独立したデータです。
以下の方法で共有できます:

**方法1: バックアップ・リストア**
1. データを持っている人がバックアップを作成
2. バックアップファイルを共有
3. 受け取った人がリストアを実行

**方法2: エクスポート・インポート**
- 各データ（インシデント、変更要求等）をCSVエクスポート
- 受け取った人が手動で登録

**将来の機能**: サーバー同期機能を検討中

## 8. セキュリティ

### Q8-1. 安全なパスワードの作り方を教えてください

**A**: 以下のルールに従ってください:

**必須要件**:
- 8文字以上
- 大文字を含む
- 数字を含む
- 特殊文字を含む（推奨）

**良いパスワードの例**:
- `Appsuite2026!` ✅
- `MyP@ssw0rd2026` ✅

**悪いパスワードの例**:
- `password` ❌（短い、大文字・数字なし）
- `12345678` ❌（数字のみ）
- `appsuite` ❌（小文字のみ）

**ヒント**:
- 複数の単語を組み合わせる
- 記号を挟む
- 他のサービスと同じパスワードを使わない

### Q8-2. パスワードの有効期限はありますか？

**A**: デフォルトでは無期限ですが、セキュリティのため90日ごとの変更を推奨します。

管理者は有効期限を設定できます（Phase 5で実装予定）。

### Q8-3. 他の人に自分のアカウントを使わせてもよいですか？

**A**: **絶対にしないでください**。

**理由**:
1. セキュリティリスク
2. 監査ログで誰が操作したか不明になる
3. 不正操作の責任が問われる可能性

**代わりに**: 新しいユーザーアカウントを作成してください（管理者に依頼）。

### Q8-4. 監査ログは誰が見られますか？

**A**:

| 項目 | 管理者 | 一般ユーザー |
|------|--------|------------|
| 全ユーザーの操作ログ | ✅ 可能 | ❌ 不可 |
| 自分の操作ログのみ | ✅ 可能 | ✅ 可能 |

監査ログはセキュリティ監視・コンプライアンス対応のため記録されています。

### Q8-5. セキュリティ上、気をつけることは何ですか？

**A**: 以下のベストプラクティスに従ってください:

**パスワード管理**:
- [ ] 他人に教えない
- [ ] 定期的に変更する（90日ごと推奨）
- [ ] 他のサービスと同じパスワードを使わない
- [ ] 複雑なパスワードを設定する

**セッション管理**:
- [ ] 離席時はログアウトする
- [ ] 共用PCでは「シークレットモード」を使う
- [ ] 無操作30分でタイムアウト

**データ管理**:
- [ ] 重要な変更前にバックアップを取る
- [ ] 定期的にバックアップを保存する
- [ ] 削除は慎重に行う

**その他**:
- [ ] HTTPSでアクセス（本番環境）
- [ ] 最新のブラウザを使用
- [ ] 不審な操作があれば管理者に報告
