## 1. 文書概要

### 1.1 目的
本文書は、DeskNet's Neo AppSuite管理運用システムの詳細要件を定義し、開発・テスト・運用の基準となる仕様を明確化することを目的とする。

### 1.2 対象読者
- プロジェクトマネージャー
- システムアーキテクト
- 開発エンジニア
- テストエンジニア
- 運用担当者
- システム管理者

### 1.3 用語定義

| 用語 | 定義 |
|------|------|
| AppSuite | DeskNet's Neoのノーコード業務アプリ作成機能 |
| インシデント | アプリの障害・不具合・問い合わせ事象 |
| 変更要求 | アプリの機能追加・改善・修正の申請 |
| 監査ログ | システム操作の記録 |
| API | Application Programming Interface |

## 2. システム概要

### 2.1 システム名称
**AppSuite管理運用システム**（DeskNet's Neo連携）

### 2.2 システム目的
DeskNet's Neo AppSuiteで作成された業務アプリケーションを一元的に管理・運用するためのWebベースの管理システムを提供する。

## 3. 機能要件

### 3.1 ダッシュボード機能

#### 3.1.1 概要
システム全体の状況を一目で把握できる統計情報表示機能。

#### 3.1.2 要件一覧

| 要件ID | 要件名 | 詳細 | 優先度 |
|--------|--------|------|--------|
| DASH-001 | 統計カード表示 | ユーザー数、アプリ数、インシデント数、変更要求数を表示 | 必須 |
| DASH-002 | リアルタイム更新 | 設定間隔で自動更新 | 必須 |
| DASH-003 | 最近の操作ログ | 直近の操作履歴を時系列表示 | 必須 |
| DASH-004 | アプリ一覧サマリ | 稼働中アプリの概要表示 | 必須 |

#### 3.1.3 統計情報項目

| 項目名 | 説明 | データソース |
|--------|------|-------------|
| 総ユーザー数 | 登録されている全ユーザー数 | users |
| 有効ユーザー | ステータスが「有効」のユーザー数 | users (status=active) |
| アプリ数 | 登録されている全アプリ数 | apps |
| 稼働中アプリ | ステータスが「稼働中」のアプリ数 | apps (status=active) |
| 未解決インシデント | ステータスが「クローズ」以外のインシデント数 | incidents (status≠closed) |
| 保留中の変更 | ステータスが「承認待ち」の変更要求数 | changes (status=pending) |

### 3.2 ユーザー管理機能

#### 3.2.1 概要
- **目的**: AppSuiteを利用するユーザーのアカウント登録・編集・削除と権限設定を一元管理する
- **意味**: 誰がどの権限でシステムにアクセスできるかを可視化し、適切なアクセス制御を実現する
- **必要性**: セキュリティ確保・不正アクセス防止・監査対応のために不可欠な機能

#### 3.2.2 要件一覧

| 要件ID | 要件名 | 詳細 | 優先度 |
|--------|--------|------|--------|
| USER-001 | ユーザー一覧表示 | 登録ユーザーをテーブル形式で表示 | 必須 |
| USER-002 | ユーザー検索 | ユーザー名・メールアドレスでの検索 | 必須 |
| USER-003 | フィルタリング | ステータス・権限でのフィルタ | 必須 |
| USER-004 | 新規登録 | ユーザー情報の新規登録 | 必須 |
| USER-005 | 編集 | 既存ユーザー情報の編集 | 必須 |
| USER-006 | 削除 | ユーザーの削除（論理削除） | 必須 |
| USER-007 | ページネーション | 大量データ対応のページング | 必須 |
| USER-008 | DeskNet's Neo連携 | APIによるユーザー同期 | 任意 |

#### 3.2.3 ユーザーデータ項目

| 項目名 | データ型 | 必須 | 説明 |
|--------|---------|------|------|
| id | String | ○ | ユーザーID（自動採番） |
| username | String | ○ | ユーザー名 |
| email | String | ○ | メールアドレス |
| department | String | - | 部署名 |
| role | Enum | ○ | 権限（管理者/ユーザー） |
| status | Enum | ○ | ステータス（有効/無効） |
| lastLogin | DateTime | - | 最終ログイン日時 |
| createdAt | DateTime | ○ | 作成日時 |
| updatedAt | DateTime | ○ | 更新日時 |

### 3.3 アプリ管理機能

#### 3.3.1 概要

| APP-004 | ステータスフィルタ | 稼働状況によるフィルタリング | 必須 |
| APP-005 | 新規登録 | アプリ情報の新規登録 | 必須 |
| APP-006 | 編集 | 既存アプリ情報の編集 | 必須 |
| APP-007 | 削除 | アプリの削除 | 必須 |

#### 3.3.3 アプリデータ項目

### 3.5 変更管理機能

#### 3.5.1 概要
- **目的**: アプリの機能追加・改善・修正要求を登録し、承認フローを通じて計画的に変更を実施する
- **意味**: 変更履歴を記録し影響範囲を把握することで、無計画な変更によるリスクを低減する
- **必要性**: 変更の追跡・承認管理・ロールバック対応のために必要不可欠な管理プロセス

| CHG-004 | ステータスフィルタ | 承認状況によるフィルタリング | 必須 |
| CHG-005 | 新規登録 | 変更要求の新規登録 | 必須 |
| CHG-006 | 編集 | 変更要求の編集 | 必須 |
| CHG-007 | 承認フロー | 承認/却下の処理 | 必須 |

#### 3.5.3 変更要求データ項目

### 3.6 監査ログ機能

- **意味**: 誰がいつ何をしたかを追跡可能にし、セキュリティ監査やトラブル発生時の原因調査に活用する
- **必要性**: コンプライアンス対応・内部統制・情報セキュリティ管理の観点から必須の機能

#### 3.6.2 要件一覧

## 5. 制約事項

### 5.1 技術的制約
- フロントエンド: HTML5, CSS3, Vanilla JavaScript
- データストレージ: localStorage（ブラウザ内蔵）
- 外部依存: Font Awesome 6.4.0（CDN）

### 5.2 業務的制約
- DeskNet's Neoのライセンス契約が必要
- システム管理者権限でのAPI利用が必要

### 5.3 前提条件
- モダンブラウザ環境での使用
- JavaScript有効化
- インターネット接続（CDN利用のため）
